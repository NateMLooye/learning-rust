Reference: Vim Keybind Cheatsheet

Not vim-from-zero — the motions and operators that actually pay off when stub-filling Rust and reading dense backend source.

You already know the basics (movement, insert mode, saving). This page is scoped to what's genuinely useful for this course's shape of work: replacing a single todo!() with real logic, filling one match arm at a time, and jumping around unfamiliar files without losing your place. Each lesson also has a small Vim reminder-style dropdown tying one of these to that lesson's actual edit — this page is the durable version to come back to. Every command below is a dropdown: the line always shown is the command and a one-line gloss, click to expand the "where you'll actually use it" detail.

Undo & redo — your safety net

Every exercise here means typing real logic yourself — filling in a body early on, sometimes designing a signature or trait later — which means you will occasionally delete more than you meant to. These two commands make that free to recover from — use them liberally instead of second-guessing a ci{ or dap.

u — undo the last change

Repeatable — hit it again to keep undoing further back. Where you'll use it: any time an edit didn't land the way you expected, before you start manually retyping to fix it.

Ctrl-R — redo (undo the undo)

Steps back forward through the undo history. Where you'll use it: you pressed u one too many times, or undid something to inspect the "before" state and now want the edit back.

Text objects & editing operators — the actual point of this page

Rust code is far more delimiter-heavy than Python (braces, parens, generic angle brackets). Vim's operator + text object combo is built exactly for this: an operator (change, delete, yank) plus a text object (inner or around something) targets precisely the text between a pair of delimiters, without you counting characters or visually selecting.

ciw — change the word under the cursor

change inner word — deletes the word under the cursor and drops you into insert mode. Where you'll use it: renaming a variable or parameter name in place.

cw — change from the cursor to the end of the word, not the word plus trailing space

change word — vim special-cases this one (:help cw): unlike dw/yw, which also consume the whitespace up to the start of the next word, cw stops at the end of the current word when the cursor is on a non-blank character, behaving like ce. It's an exception to the general operator+text-object grammar below, not a strict instance of it. Where you'll use it: fixing a typo partway through an identifier or a doc-comment word without retyping its start.

ci( / ci) — change everything inside the nearest parens

change inner (parens) — delimiters untouched. Where you'll use it: rewriting a function's argument list, or the inside of a call like add(2, 3).

ci{ / ci} — change everything inside the nearest braces

change inner {braces} — the single most useful one here. Where you'll use it: replacing a whole function body (todo!() and everything else) or one match arm's block in one motion.

ci< / ci> — change everything inside the nearest angle brackets

change inner <angle brackets>. Where you'll use it: generic parameter lists (<T: Display>) and lifetime parameter lists (<'api>) are both angle-bracket text objects — rewrite the bound or rename the lifetime without retyping the brackets.

ci" — change the contents of the nearest quoted string

change inner "quotes" — quotes untouched. Where you'll use it: editing a string literal (test fixture data, an error message) without retyping the quotes.

di{ / da{ — delete inside/around the nearest braces

delete instead of change — i leaves the braces, a removes them too. Where you'll use it: deleting a whole match arm's block including or excluding the braces.

yi( — yank (copy) everything inside the nearest parens

yank inner (parens) — without entering insert mode. Where you'll use it: copying an argument list to reuse elsewhere.

yap / dap — yank/delete around the whole paragraph (function)

yank/delete around paragraph — blank lines act as paragraph boundaries in Rust, so this targets a whole function at once. Where you'll use it: moving an entire function to a new file during a module-split refactor: yap in the old file, switch buffers, p to paste, dap back in the original once it's safely moved.

. — repeat the last change exactly

Not a mnemonic — just the repeat-last-change symbol. Where you'll use it: filled one match arm with ci{...<Esc>? Move to the next arm and hit . for the same shape of edit.

Reading the pattern: every command above except the last is operator (c/d/y) + scope (inner or around) + text object (a delimiter pair, a word, or paragraph). Once that grammar clicks, you can guess a command you've never used before — di", ya(, ci[ — instead of memorizing each one as an unrelated fact.

Jumps & search

% — jump to the matching bracket/paren/brace

Bouncing between an opening { and its closing } in a nested match or a multi-field struct-shaped enum variant (Lesson 11's Parse { ... }) — genuinely more load-bearing in Rust than it ever was in Python.

* / # — search forward/backward for the word under the cursor

Whole-word search. Where you'll use it: "where else in this file is expr used" — the vim-native version of what gr (LSP references) does across files.

{ / } — jump to the previous/next blank line

Paragraph motion. Blank lines separate functions in idiomatic Rust, so this is a fast way to hop from one function/impl block to the next in a long backend source file.

gg / G — jump to the first/last line of the file

Where you'll use it: getting to the top of a field-study file fast, or to the bottom to see the last impl block.

Ctrl-o / Ctrl-i — jump backward/forward through the jumplist

Where you'll use it: after gd (go to definition) jumps you into a different file or a faraway line, Ctrl-o gets you straight back to where you were — don't scroll back by hand.

Editing efficiency for stub-fill work

o / O — open a new line below/above and enter insert mode

Where you'll use it: adding a new match arm or a new field without splitting an existing line by hand.

Ctrl-v — visual block mode, select a rectangular column of text

Where you'll use it: aligning several match arms, or adding the same prefix (e.g. pub ) to a column of lines at once.

>> / << — indent/outdent the current line one shiftwidth

Rarely needed here — format-on-save (Lesson 2) handles real indentation — but useful mid-edit before a save.

The one habit worth building deliberately

When you catch yourself about to arrow-key or hjkl-repeat to a delimiter, stop and ask "is there a text object for this?" almost always yes. The payoff compounds specifically in Rust because of how much of the language's structure lives inside {}/()/<> pairs — far more than typical Python.

Vim's own :help motion.txt (mirrored at vimhelp.org) — section 6, "Text object selection," is the canonical reference for every ci(/di{/yaw-style command above. Also reachable without leaving the editor: :help text-objects.
